# User Permissions API<a id="user-permissions-api"></a>

## Overview<a id="overview"></a>

The User Permissions API provides endpoints to manage users and their permissions within your application. It allows you to list users, retrieve and update their permissions, and manage available permission types. This API is ideal for applications that require fine-grained access control and user management.

## Endpoints<a id="endpoints"></a>

### 1. List Users<a id="1-list-users"></a>

- **Endpoint:** `GET /users`
- **Description:** Retrieves a list of all users.
- **Request Example:**
  ```http
  GET {{baseUrl}}/users
  ```
- **Response Example:**
  ```json
  [
    { "id": 1, "name": "Alice", "email": "alice@example.com" },
    { "id": 2, "name": "Bob", "email": "bob@example.com" }
  ]
  ```

### 2. Get User Permissions<a id="2-get-user-permissions"></a>

- **Endpoint:** `GET /users/{userId}/permissions`
- **Description:** Retrieves the permissions assigned to a specific user.
- **Request Example:**
  ```http
  GET {{baseUrl}}/users/1/permissions
  ```
- **Response Example:**
  ```json
  [
    { "permission": "read", "granted": true },
    { "permission": "write", "granted": false }
  ]
  ```

### 3. Update User Permissions<a id="3-update-user-permissions"></a>

- **Endpoint:** `PUT /users/{userId}/permissions`
- **Description:** Updates the permissions for a specific user. Provide an array of permission objects.
- **Request Example:**

  ```http
  PUT {{baseUrl}}/users/1/permissions
  Content-Type: application/json

  [
    {"permission": "read", "granted": true},
    {"permission": "write", "granted": true}
  ]
  ```

- **Response Example:**
  ```json
  [
    { "permission": "read", "granted": true },
    { "permission": "write", "granted": true }
  ]
  ```

### 4. List Permissions<a id="4-list-permissions"></a>

- **Endpoint:** `GET /permissions`
- **Description:** Lists all available permission types in the system.
- **Request Example:**
  ```http
  GET {{baseUrl}}/permissions
  ```
- **Response Example:**
  ```json
  [{ "permission": "read" }, { "permission": "write" }, { "permission": "admin" }]
  ```

### 5. Grant User Permission<a id="5-grant-user-permission"></a>

- **Endpoint:** `POST /users/{userId}/permissions`
- **Description:** Grants a specific permission to a user. Provide the permission object in the request body.
- **Request Example:**

  ```http
  POST {{baseUrl}}/users/2/permissions
  Content-Type: application/json

  {"permission": "admin", "granted": true}
  ```

- **Response Example:**
  ```json
  [{ "permission": "admin", "granted": true }]
  ```

## Error Handling<a id="error-handling"></a>
